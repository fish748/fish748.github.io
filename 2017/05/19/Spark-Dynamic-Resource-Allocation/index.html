Spark Dynamic Resource Allocation

　　Spark 从 1.5 版本开始引入动态资源分配 (Dynamic Resource Allocation) 机制，在任务运行过程中动态申请和释放资源，适合在一个集群中同时运行多个任务的场景。

比较

  • 默认资源分配：一次性向集群申请全部需要的资源，获得资源后，直到任务运行完毕不会把资源释放。
  • 动态资源分配：一开始只向集群申请部分资源，根据任务资源需求量在任务运行中向集群申请更多资源；在任务执行过程中，会向集群返还部分闲置资源。

资源分配策略

　　资源分配的基本单位是 Executor，当任务需要资源时，向集群申请更多的 Executor；当任务中有闲置资源时，向集群返还 Executor。

资源获取

  • 获取条件：有在 pending 的 task，并且 pending 了一段时间。
  • 当有 task 保持 pending 状态 spark.dynamicAllocation.schedulerBacklogTimeout 秒后，资源申请被触发；此后，每隔 spark.dynamicAllocation.sustainedSchedulerBacklogTimeout 秒，Spark 都会检查任务中是否还有 pending 的 task，如果有则再次触发资源申请。
  • Executor 申请的数量以指数增长，即 1、2、4、8……。

资源释放

  • 释放条件：Executor 闲置了一段时间。
  • 当有 Executor 闲置了 spark.dynamicAllocation.executorIdleTimeout 秒后，该 Executor 的资源将被返还给集群。

External shuffle service

　　在 shuffle 时，每个 Executor 需要把上一个 stage 的 mapper 输出写入到磁盘，然后作为 server 等待下一个 stage 的 reducer 来获取 map 的输出。因而，若 Executor 在 map 阶段完成后被释放，reducer 将无法找到 block 的位置。

　　 因此，在使用 Dynamic Resource Allocation 时，必须开启 external shuffle service。开启后，mapper 的输出位置将会由该 service 来记录，Executor 不需要再保留状态信息，可以被安全释放。

开启 Dynamic Resource Allocation

external shuffle service 配置

  • 将 Spark 自带的 $SPARK_HOME/yarn/spark-2.11-yarn-shuffle.jar 添加到 YARN NodeManager 的 classpath 中。
  • 在 yarn-site.xml 中将 spark_shuffle 添加到 yarn.nodemanager.aux-services，并将 yarn.nodemanager.aux-services.spark_shuffle.class 设置为 org.apache.spark.network.yarn.YarnShuffleService。
  • 在 etc/hadoop/yarn-env.sh 中设置 YARN_HEAPSIZE，增加 NodeManager 的堆大小，尽量避免在 shuffle 过程中发生 GC。

Spark 配置

  • 使用 external shuffle service：设置 spark.shuffle.service.enabled 为 true
  • 使用 dynamic resource allocation：设置 spark.dynamicAllocation.enabled 为 true
  • 任务启动时的初始 Executor 数量：spark.dynamicAllocation.initialExecutors

其他配置

  • 最大可申请的 Executor 数量：spark.dynamicAllocation.maxExecutors
  • 最小保留的 Executor 数量：spark.dynamicAllocation.minExecutors
  • 检查 pending tasks 的时间间隔 (秒)：spark.dynamicAllocation.schedulerBacklogTimeout
